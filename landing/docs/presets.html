Rule Presets

Presets are pre-configured sets of rules optimized for specific site types. Select a preset before running an analysis to focus on the most relevant checks.

Using Presets

In the Web UI

Before clicking "Analyze", select a preset from the dropdown. You can further customize by toggling individual rules on or off.

In the SDK (CLI)

Use the --only flag with the rule keys included in a preset:

# Run only essential checks
seo-crawler --mode=full --url=http://localhost:3000 \
  --only="title_exists,meta_description_exists,h1_exists,canonical_exists,robots_txt_exists,sitemap_exists,uses_https,http_status_ok"

In the config file

# .seo-crawler.yml
mode: full
url: http://localhost:3000
only:
  - title_exists
  - meta_description_exists
  - h1_exists
  - canonical_exists
  - http_status_ok
  - uses_https

Preset Details

Full Scan

Runs all rules across all categories. Use this for a complete picture of your site's health. Best for initial audits or periodic comprehensive checks.

Essential

The minimum set of rules every website should pass. Covers basic on-page SEO, critical technical issues, and security basics.

Blog / Article

Optimized for content-heavy sites. Emphasizes heading structure, content quality metrics, author signals (E-E-A-T), structured data for articles, and social sharing metadata.

E-Commerce

Tailored for product pages and transactional sites. Focuses on product schema, canonical URLs (to handle pagination/filters), performance, security, and mobile usability.

GEO / AI Readiness

All 24 GEO rules to ensure your content is optimized for AI search engines. Checks AI crawler access, llms.txt, content citability, entity authority, and AI-friendly content structure.
