Rules / Performance / HTML document size is reasonable

HTML document size is reasonable

medium

html_size

Checks that the HTML document size is reasonable.

What does this rule check?

The raw HTML document size should be reasonable (under 100KB for most pages). Excessively large HTML indicates inline data, embedded SVGs, or server-rendered content that should be optimized.

Why is this important?

Large HTML documents take longer to download and parse. They delay DOM ready time and increase memory usage, especially on mobile devices.

How to fix

Move inline styles/scripts to external files. Paginate long lists. Use lazy-loading for dynamic content.

Examples

Incorrect 
<!-- 500KB HTML document with embedded SVGs and inline data -->
Correct 
<!-- 30KB HTML document with external assets -->
