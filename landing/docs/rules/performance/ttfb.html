Rules / Performance / Time to First Byte is fast

Time to First Byte is fast

high

ttfb

Checks that Time to First Byte is fast.

What does this rule check?

TTFB measures the time from the request to the first byte of the response. It reflects server processing time, network latency, and caching effectiveness. Target is under 800ms, ideally under 200ms.

Why is this important?

Slow TTFB delays everything — the page cannot start rendering until the first byte arrives. It's a direct Core Web Vitals metric and affects user-perceived performance.

How to fix

Optimize server-side processing, enable caching, use a CDN, and optimize database queries.

Examples

Incorrect 
TTFB: 3.2 seconds
<!-- Server is slow to respond -->
Correct 
TTFB: 150ms
<!-- Fast server response with caching -->
