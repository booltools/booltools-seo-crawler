JSON-LD is valid JSON

high

jsonld_valid

Checks that JSON-LD is valid JSON syntax.

What does this rule check?

JSON-LD must be valid JSON that follows the Schema.org vocabulary. Invalid JSON (missing quotes, trailing commas, etc.) means search engines cannot parse your structured data at all.

Why is this important?

Invalid JSON-LD is completely ignored by search engines. Your structured data provides zero benefit if it can't be parsed.

How to fix

Validate your JSON-LD using a JSON validator and Google's Rich Results Test tool.

Examples

Incorrect 
<script type="application/ld+json">
{@context: "https://schema.org", @type: Article,}
</script>
<!-- Invalid: unquoted keys, trailing comma -->
Correct 
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article"
}
</script>
