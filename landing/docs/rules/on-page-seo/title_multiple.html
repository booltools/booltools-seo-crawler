Page has only one title tag

high

title_multiple

Last updated: 2026-06-24

Checks that the page has only one <title> tag.

What does this rule check?

HTML specification requires exactly one <title> tag per document. Multiple title tags confuse search engines about which title to display. This often happens with server-side rendering frameworks that inject titles at multiple levels, or when a CMS adds a title and the theme adds another.

Why is this important?

Multiple title tags confuse search engines about which title to use for the page. Browsers will typically use the first one, while search engines may use any of them or merge them unpredictably.

How to fix

Remove duplicate <title> tags, keeping only one in the <head>. Check your template hierarchy to ensure only one component is responsible for setting the title.

Examples

Incorrect 
<head>
  <title>My Page - Site Name</title>
  <title>Another Title</title>
  <!-- Two title tags! -->
</head>
Correct 
<head>
  <title>My Page - Site Name</title>
  <!-- Only one title tag -->
</head>
