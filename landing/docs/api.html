API Reference

The backend exposes a REST API on port 8080 by default.

Endpoints

MethodEndpointDescription
POST/api/crawlStart a new crawl
GET/api/crawl/:id/progressSSE progress stream
GET/api/report/:idFull audit report
GET/api/report/:id/export/csvExport as CSV
GET/api/report/:id/export/mdExport as Markdown
GET/api/rulesList rules and presets
GET/api/healthHealth check

Start a Crawl

POST /api/crawl
Content-Type: application/json

{
  "domain": "example.com",
  "maxPages": 100,
  "selectedRules": ["title_exists", "meta_description_exists"]
}

Response:

{
  "id": "uuid-here",
  "domain": "example.com",
  "status": "pending",
  "maxPages": 100,
  "pagesCrawled": 0,
  "issuesFound": 0,
  "createdAt": "2026-06-22T12:00:00Z"
}

Progress Stream (SSE)

GET /api/crawl/:id/progress
Accept: text/event-stream

Returns a Server-Sent Events stream with real-time updates:

data: {"jobId":"...","status":"crawling","pagesCrawled":5,"totalDiscovered":12,"currentUrl":"...","pageIssues":[...]}

Get Report

GET /api/report/:id

Returns the full audit report with SEO score, GEO score, issues grouped by severity, page-by-page breakdown, and links to change.

Export Report

# CSV format
GET /api/report/:id/export/csv

# Markdown format (AI-agent-friendly)
GET /api/report/:id/export/md

List Rules

GET /api/rules

Returns all available audit rules grouped by category, including severity and description for each rule. Also returns preset definitions.

Health Check

GET /api/health

Response: {"status":"ok"}

Example: cURL

# Start a crawl
curl -X POST http://localhost:8080/api/crawl \
  -H "Content-Type: application/json" \
  -d '{"domain": "example.com"}'

# Get the report
curl http://localhost:8080/api/report/YOUR_JOB_ID

# Export as markdown
curl http://localhost:8080/api/report/YOUR_JOB_ID/export/md -o report.md
