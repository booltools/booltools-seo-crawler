Rules Reference

Complete reference for all 130+ audit rules grouped by category.

Severity Levels

SeverityDescription
CriticalMust fix — blocks indexing or causes major SEO damage
HighShould fix — significant negative impact on SEO
MediumRecommended — moderate impact, improves rankings
LowNice to have — minor improvements
InfoInformational — no direct SEO impact

On-Page SEO (15 rules)

Title tags, meta descriptions, heading hierarchy, image alt text, and content optimization.

  • title_exists — Page has a title tag
  • title_length — Title length is between 30-60 characters
  • title_unique — Title is unique across the site
  • meta_description_exists — Page has a meta description
  • meta_description_length — Meta description is 120-160 characters
  • h1_exists — Page has an H1 heading
  • h1_single — Only one H1 per page
  • heading_hierarchy — Headings follow logical order
  • img_alt_text — Images have alt attributes
  • img_dimensions — Images specify width/height
  • img_lazy_loading — Below-fold images use lazy loading

Technical SEO (24 rules)

Canonical URLs, robots directives, sitemaps, redirects, and HTTP status codes.

  • canonical_exists — Page has a canonical URL
  • canonical_valid — Canonical URL is valid and accessible
  • robots_txt_exists — Site has a robots.txt
  • robots_txt_valid — robots.txt is properly formatted
  • sitemap_exists — Site has a sitemap.xml
  • sitemap_valid — Sitemap is valid XML
  • http_status_ok — Page returns 200 status
  • no_redirect_chains — No long redirect chains
  • meta_robots_index — Page allows indexing

GEO — AI Search Readiness (24 rules)

Generative Engine Optimization checks for AI crawler access, content citability, and entity authority.

  • ai_crawler_not_blocked — AI crawlers are not blocked in robots.txt
  • llms_txt_exists — Site has llms.txt
  • llms_txt_valid — llms.txt is properly formatted
  • content_citability — Content is structured for citation
  • entity_authority — Clear entity ownership signals
  • ai_friendly_headings — Headings are descriptive for AI parsing
  • ai_friendly_structure — Content uses AI-parseable structure
  • factual_claims_sourced — Claims have supporting sources

Performance (9 rules)

Page size, response time, compression, and resource optimization.

  • page_size — Total page size under threshold
  • ttfb — Time to first byte under 600ms
  • compression — Response uses gzip/brotli
  • render_blocking_resources — No render-blocking scripts in head
  • http_requests — Reasonable number of HTTP requests

Security (9 rules)

HTTPS, security headers, and secure configuration.

  • uses_https — Site uses HTTPS
  • hsts_header — HSTS header present
  • security_csp — Content-Security-Policy header
  • security_xfo — X-Frame-Options header
  • security_xcto — X-Content-Type-Options header

Links (6 rules)

Internal linking, external links, and broken link detection.

  • internal_links_exist — Page has internal links
  • no_broken_links — No broken internal links
  • external_links_nofollow — External links use appropriate rel

Structured Data (3 rules)

JSON-LD, breadcrumbs, and schema.org validation.

  • jsonld_exists — Page has JSON-LD structured data
  • jsonld_valid — JSON-LD is valid
  • breadcrumb_schema — Breadcrumb structured data present

Social / Open Graph (12 rules)

Open Graph tags and Twitter Card metadata.

  • og_title — og:title present
  • og_description — og:description present
  • og_image — og:image present
  • twitter_card — Twitter card meta tag

Other Categories

  • Content (2 rules) — Word count, text-to-HTML ratio
  • Accessibility (5 rules) — Language, viewport, ARIA
  • Mobile (2 rules) — Viewport, touch targets
  • URL Structure (6 rules) — Lowercase, length, special characters
  • Internationalization (2 rules) — Hreflang validation
  • E-E-A-T (6 rules) — Author, about, contact, privacy pages
  • Duplicate Content (3 rules) — Duplicate titles, descriptions, body
